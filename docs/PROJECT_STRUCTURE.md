# Project Structure

## 模块职责

- `main.py`：演示入口，用于串联完整流程。
- `src/`：数据加载、清洗、格式转换。
- `analysis/`：专注度与压力分析、推荐策略、报告拼装。
- `backend/`：API、数据库、会话管理。
- `data/raw/`：原始输入数据（建议只放样例和可公开数据）。
- `data/processed/`：运行输出数据（建议不纳入版本管理）。
- `docs/`：架构图、API 说明、分工文档、答辩材料。
- `electron-dist/`：前端桌面端相关文件；打包产物应忽略。

## 新增文件落位规则

1. 新算法文件放到 `analysis/`。
2. 新 API 接口文件放到 `backend/`。
3. 新数据脚本放到 `src/`。
4. 新文档统一放到 `docs/`。
5. 任何自动生成或打包文件都应加入 `.gitignore`。

## 比赛协作建议

1. 按模块分工：算法、后端、前端、文档各自维护本目录。
2. 功能开发时优先小步提交，避免一次提交跨多个模块。
3. 合并前确认主流程可运行：`python main.py`。
